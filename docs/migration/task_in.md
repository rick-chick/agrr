## माइग्रेशन निर्माण चेकलिस्ट

### AgriculturalTask मॉडल विशेषताएं

- `name` (string, required): कार्य नाम। विशिष्टता आवश्यक (संदर्भ कार्य नाम से अद्वितीय, उपयोगकर्ता स्वामित्व उपयोगकर्ता के भीतर अद्वितीय)
- `description` (text, nullable): कार्य विवरण
- `time_per_sqm` (float, nullable): प्रति इकाई क्षेत्र (sqm) में आवश्यक समय (घंटे/sqm)
- `weather_dependency` (string, nullable): मौसम निर्भरता। मान: 'low', 'medium', 'high'
- `required_tools` (text, nullable): आवश्यक उपकरण। JSON सरणी के रूप में संग्रहीत (उदाहरण: `["कुदाल", "रोपण करने का औजार"]`)
- `skill_level` (string, nullable): कौशल स्तर। मान: 'beginner', 'intermediate', 'advanced'
- `is_reference` (boolean, default: true): संदर्भ कार्य ध्वज। मास्टर डेटा true है
- `user_id` (integer, nullable): उपयोगकर्ता ID। संदर्भ कार्यों के लिए null
- `region` (string, nullable): क्षेत्र कोड। 'jp', 'us', 'in', आदि

## マイグレーション作成時の注意点

1. **初期実装方針**: 
   - **まずはインド（region='in'）のマスタデータのみを作成**
   - 全て参照タスク（is_reference=true）として作成
   - 後で他の地域（'jp', 'us'など）を追加する際は別のマイグレーションで対応

2. **地域ごとの作成**: 各タスクは地域（'jp', 'us', 'in'）ごとに作成するか確認
   - 同じタスクでも地域によって名称や説明が異なる場合がある
   - 必要な場合は地域ごとに別レコードとして作成
   - 初期実装では'in'のみを作成

3. **参照タスクの設定**: 
   - 全てのタスクは `is_reference=true` として作成
   - `user_id` は `null`（参照タスクはシステム所有）

4. **名前の一意性**: 
   - 参照タスク（is_reference=true）は名前で一意
   - 同じ名前のタスクを複数地域で作成する場合は、regionを含めた検索条件で確認
   - 初期実装（inのみ）では、名前とis_reference=trueで一意性を確認

5. **必須属性**: 
   - `name` は必須
   - `region` は 'in' を設定
   - `is_reference` は true を設定
   - `user_id` は null を設定
   - その他の属性はnullableだが、可能な限り設定することが推奨

6. **JSON配列の扱い**: 
   - `required_tools` はJSON配列として保存（例: `["ツール1", "ツール2"]`）
   - Rubyの配列を `to_json` でシリアライズして保存

7. **既存データの確認**: 
   - マイグレーション実行前に既存のagricultural_tasksテーブルに同じ名前のタスク（region='in', is_reference=true）が存在しないか確認
   - 存在する場合は `find_or_initialize_by` を使用して更新するか、新しいレコードを作成するか検討

8. **工具について**: 
   - **重機は使用しない**前提で設定（トラクター、ロータリーティラー、大型機械などは含めない）
   - 手動工具のみを想定して`required_tools`を設定
   - `time_per_sqm`は手作業を前提とした所要時間を設定

9. **作業時間（time_per_sqm）について**: 
   - 単位は**時間/㎡**で保存
   - 手作業を前提とした現実的な値を設定
   - 実際の作業効率は作物の種類、作業者のスキル、天候などによって大きく変動するため、目安として設定
   - マイグレーション時に地域や文化によって異なる作業効率を考慮して調整することを推奨（初期実装ではインドの標準的な値を設定）
