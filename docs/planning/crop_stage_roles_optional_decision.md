# crop_stage_roles（案A）— 見送り記録

**判定日**: 2026-04-15  
**結論**: **案A（DB の `crop_stage_roles` 導入）は採用しない**（MVP では `StageRoleResolver` の既存ヒューリスティクスで十分）。

**根拠**: エントリ作物スケジュールの参照データにおいて、まき／植え相当ステージの欠損による表示破綻はテストデータ上発生しておらず、追加テーブルより複製ステージ＋名前パターン解決を優先した。

**再検討条件**: 参照作物マスタで「定植」明示が無い品目が増え、誤った植えステージ解決が頻発する場合。
