# 作物スケジュール（家庭菜園・エントリ）企画要件

**ステータス**: ドラフト  
**最終更新**: 2026-04-15  

---

## 1. 背景・目的

### 1.1 背景

- 家庭菜園ユーザーは「いつ植え／いつまき／その先」を、文献・検索・経験則に分散して調べることが多い。
- 地域と最新の気象・予測に基づくと時期判断の納得感が上がる一方、作物ごとの知識と結びつけて一覧で比較する体験は不足しがちである。

### 1.2 目的

- **選択地域の最新気象データ・予測**を前提に次を実現する。
  - **一覧**：**植え時・まき時**と**その後のざっくりスケジュール**が比較でき、**どのタイミングに何を始めるか**を検討できること。
  - **詳細**：その作物について、**今後どんな作業がいつ頃必要か**がざっくりわかること。

### 1.3 成功時のユーザ価値

- **迷いの削減**（特に週末の段取り・購入のタイミング）。
- **地域に合わせた目安**（固定カレンダーだけではない納得感）。

---

## 2. 対象ユーザー・前提

| 項目 | 内容 |
|------|------|
| 主対象 | **エントリ〜中級の家庭菜園ユーザー**（プロ農業向け最適化は本要件の主目的としない）。 |
| 前提知識 | **植え時／まき時**に加え、**口語ラベル**（例：種をまく目安／畑に植える目安）を併示し、用語負荷を下げる。 |
| デバイス | **スマホ優先**（一覧の比較性・詳細の縦スクロールを前提）。 |

### 2.1 ユーザ意図（整理）

- **定植と直播**はユーザ意図として**同一（「始める」）**。画面でモードを分けて選ばせない。
- 一方、**情報として**は **植え時** と **まき時** の両方を一覧に示し、計画のための比較に使う。

---

## 3. スコープ

### 3.1 対象範囲（MVP）

- **地域選択**：**ドロップダウンのみ**（検索・現在地は必須としない）。
- **作物一覧**（地域・気象・予測に基づく表示）。
- **作物詳細**：**一覧から開く詳細は1画面**。カタログ参照と「育てている（マイ作物）」は**表示優先度の切替**で対応し、**専用の別詳細画面は作らない**。
- **始める／育てている**の**主問いの切替**（セグメント等）。**設定アプリ化しない**。

### 3.2 対象外（MVP）

- **設定トップ**に相当する複雑な設定群。
- **複数作物を1枚のカレンダーに重ねる専用画面**（将来検討）。
- **定植／直播を別モードでユーザに選ばせる**こと。

---

## 4. 機能要件

### 4.1 地域（FR-LOC）

| ID | 要件 |
|----|------|
| FR-LOC-01 | ユーザーは**ドロップダウン**から**地域**を選択できる。 |
| FR-LOC-02 | 地域未選択時、一覧・詳細の計算に必要な表示は**ブロック**または**誘導**し、**誤った地域前提の断定**をしない。 |
| FR-LOC-03 | 地域変更後、**一覧・詳細の表示は選択地域の気象・予測に基づき再計算**される。 |

### 4.2 一覧（FR-LIST）

| ID | 要件 |
|----|------|
| FR-LIST-01 | 一覧の各行は**作物**を表し、**選択地域**に紐づく。 |
| FR-LIST-02 | 各行に**植え時**に相当する情報と**まき時**に相当する情報を**比較可能な形**で表示する（**ユーザに定植／直播モードを選ばせない**）。 |
| FR-LIST-03 | **その後のざっくりスケジュール**（育苗→定植→収穫等の流れ）が、**一覧レベルで把握**できること。情報過多になる場合は**段階開示**（折りたたみ・タップ展開・デフォルト列の抑制）を設計に含める。 |
| FR-LIST-04 | 表示内容は**選択地域の最新気象データおよび予測**に基づき算出されること（算出根拠の開示は§4.5）。 |
| FR-LIST-05 | **エントリ向け**に、用語に**口語ラベル**を併記できること（例：植え時＝畑に植える目安、まき時＝種をまく目安）。 |
| FR-LIST-06 | デフォルトの並びは**「いま計画しやすい順」**等、**1ルールに定義**できること（仕様パラメータ）。 |

### 4.3 詳細（FR-DET）

| ID | 要件 |
|----|------|
| FR-DET-01 | **一覧から遷移する作物詳細は1画面**とし、**カタログ参照**と**育てている（マイ作物）**の**表示優先度**を切り替えられること（別詳細画面は作らない）。 |
| FR-DET-02 | 詳細では、その作物について**今後必要になる作業**が**時系列でざっくり**把握できること。 |
| FR-DET-03 | 詳細の表示も**選択地域の最新気象データおよび予測**に基づくこと。 |
| FR-DET-04 | **4フェーズ**（播種／育苗／定植／収穫等）を**グラフィカル**に表現できること（一覧より詳細な分解）。 |
| FR-DET-05 | **目安であること**、**未入力時は一般目安**であること等、**誤解防止の注記**を表示できること。 |

### 4.4 始める／育てている（FR-MODE）

| ID | 要件 |
|----|------|
| FR-MODE-01 | **始める**：新規に何をいつ始めるかの**検討**が主目的。一覧・「今日の一手」等はこの文脈に合わせる。 |
| FR-MODE-02 | **育てている**：すでに手元・圃場にある作物の**次の作業**が主目的。詳細では**次の作業を上段**にできること（データが揃う範囲で）。 |
| FR-MODE-03 | **マイ作物**の登録は**最小項目**でよい（作物・開始日等。細部は別仕様）。 |

### 4.5 透明性・信頼（FR-TRUST）

| ID | 要件 |
|----|------|
| FR-TRUST-01 | 主要な推奨・帯の変化について、**なぜその表示になるか**を**短い説明**（1行〜折りたたみ）で示せること。 |
| FR-TRUST-02 | 利用する**気象・予測の出所**または**更新の前提**をユーザーが追えること（過剰な専門用語は避ける）。 |

---

## 5. 非機能要件（NFR）

| ID | 内容 |
|----|------|
| NFR-01 **正確性の表現** | 農業結果を保証しない。**目安・リスク**として表現する。 |
| NFR-02 **更新** | 気象・予測の取得失敗時、**前回値／エラー表示／再試行**の方針を定める。 |
| NFR-03 **パフォーマンス** | 一覧の初回表示が**許容時間内**（目標値は別途）。 |
| NFR-04 **アクセシビリティ** | 色のみに依存しない（アイコン・文言補助）。 |

---

## 6. データ・ルール前提（企画レベル）

| 項目 | 内容 |
|------|------|
| 作物マスタ | 作物ごとに、**まき・植え・生育・作業**に関する**ルール可能な属性**を持てること。 |
| 地域 | ドロップダウンで選べる**地域マスタ**と、気象・予測API等との**対応**が定義できること。 |
| 計算 | **地域×気象・予測×作物ルール**から、一覧・詳細の**帯・文言・並び**を生成する。 |

※ 数式・閾値は**別：ロジック仕様**で管理する。

---

## 7. 制約・非対応（明示）

- **圃場の微気候・日当たり・土壌**はMVPでは**個別入力しない**（将来拡張）。
- **品種までの完全最適化**は主目的としない（可能なら将来）。

---

## 8. 成功指標（KPI案）

| 指標 | 例 |
|------|-----|
| 初回 | 地域選択完了率、一覧到達率 |
| 継続 | 週次再訪、詳細閲覧深度 |
| 価値 | 「迷いが減った」に相当する行動（地域変更・作物追加・再閲覧） |

---

## 9. リスクと対応方針（企画）

| リスク | 方針 |
|--------|------|
| 予報変化で表示が変わり不信 | **FR-TRUST**（理由1行・出所） |
| エントリに情報過多 | **段階開示・口語併記・並びの単純化** |
| 差別化が弱い | 訴求を**「気象単体」ではなく「作物×地域×予測×比較」**に固定 |

---

## 10. 成果物マップ（企画→実装）

| 成果物 | 内容 |
|--------|------|
| 本要件 | 合意・スコープ・FR/NFR |
| 画面要件 | ワイヤー・コンポーネント単位の振る舞い |
| ロジック仕様 | 閾値、帯の定義、予報の取り込み |
| データ定義 | 地域・作物・マイ作物 |

---

## 11. MVP の必須セット（レビュー用）

優先して合意するなら次を **Must** とする想定。

- FR-LIST-02〜04（植え／まき・ざっくりスケジュール・気象・予測ベース）
- FR-DET-02〜04（今後の作業のざっくり・気象・予測・4フェーズ表現）
- FR-LOC-01〜03（ドロップダウン地域・未選択・再計算）
- FR-TRUST-01（表示理由の短い説明）
