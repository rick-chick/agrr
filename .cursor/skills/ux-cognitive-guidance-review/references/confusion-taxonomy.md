# 迷いの分類（認知・導線レビュー）

レビュー時は必ず **1 つ以上のコード**を付ける。

## A — 何をすべきかわからない

- 次の一手が見えない
- 空画面に CTA がない
- 複数同等ボタンで優先が不明

**観測**: 画面で止まる、戻る、離脱

## B — 用語・概念がわからない

- ラベルは読めるが意味が不明
- 専門用語（GDD、作業予定同期、interaction rule 等）に補助がない

**観測**: 誤入力、ランダムクリック、ヘルプを探す

## C — 状態・結果がわからない

- 処理中か失敗か成功か区別できない
- 進捗・残り時間がない

**観測**: リロード、同操作の繰り返し

## D — なぜできないかわからない

- ボタン disabled の理由がない
- 汎用エラー（「処理に失敗しました」）のみ

**観測**: 連打、フォーム試行錯誤

## E — どこにいるかわからない

- パンくず・見出し・ナビ active の不一致
- ハブと子画面の関係が不明

**観測**: ブラウザ戻る、迷子ナビ

## F — 期待と違う

- ログイン後の着地が意図と異なる
- 保存されない、別画面が出る

**観測**: 問い合わせ、再操作

## G — 信頼できない

- 削除の影響範囲不明
- Undo なしの破壊的操作

**観測**: 操作回避、過剰確認

## 導線層 L0–L4

| 層 | 定義 |
|----|------|
| L0 | 画面内で自己解決（説明・ラベル・例） |
| L1 | 同一フロー内誘導（空状態 CTA、バナー） |
| L2 | アプリ内別画面（ナビ、マスタ登録リンク） |
| L3 | 学習リソース（research） |
| L4 | 人・サポート（お問い合わせ） |

**適切な導線**: 迷いの種類に対し、L0 を優先しつつ不足分を L1–L4 で補い、復帰可能であること。
