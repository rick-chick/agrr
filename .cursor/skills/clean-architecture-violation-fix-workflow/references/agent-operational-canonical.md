# エージェント運用手順の集約

backlog 同期・本番断定・シェル完了の境界をここに集約する。各 `.mdc` の短い原則に同趣旨を繰り返さず、判定可能な条件としてここへ寄せる。

## 洗い出し

洗い出しの手順は [`SKILL.md`](../SKILL.md) セクション0「洗い出し」が定める。`ARCHITECTURE.md` の規約と照らし、コードベースを調査して気になる点を洗い出す。`ARCHITECTURE.md` を単一ソースとし、SKILL 側に禁止事項の正解像を二重定義しない。

`git diff` / `git status` だけで棚卸しを置き換えない。

<a id="backlog-file"></a>

### Backlog ファイル運用

CA 違反 backlog は `docs/ca-violations-backlog.md` の 1 ファイルで管理する。一覧は **未修正の逸脱のみ**（計画期限・「後で直す」行は置かない）。差分起因の新規違反はセクション4 ゲート 2 回目（`git diff` 意味読み）でも拾う。

#### ライフサイクル

1. **生成**: backlog が無い／空のとき、SKILL「洗い出し」を実行し、検出した逸脱を修正単位として書き出す。逸脱ゼロなら backlog ファイルを置かない／削除して外側ループ終了へ。
2. **消費**: セクション0 で先頭の修正単位を今回のスコープに固定する。
3. **セクション6（コミット後）**: **洗い出し**を再度実行し、キューをその結果で更新する。解消した修正単位を取り除き、ゲート 2 回目で検出した新規違反、洗い出しで見つかった逸脱を追記する。**コミットのたびに洗い出しし backlog を再同期する**（洗い出しを省略しない）。
4. **空到達時の裏取り**: backlog が空になったら洗い出しを実行する。検出ゼロなら backlog を削除して [`SKILL.md`](../SKILL.md)「外側ループの終了」へ。検出があれば backlog に書き戻してセクション1 へ。

#### 最低フォーマット

```markdown
# CA Violations Backlog

最終洗い出し: <ISO8601> / 直近裏取り: <ISO8601 or none>

## 修正単位

- [ ] **<修正単位名>** — ARCHITECTURE.md 禁止 <番号> @ <層>
  - 主なファイル: `path/to/file.rb`, ...
```

#### 制約

- **backlog に未処理があるあいだも洗い出しを省略しない**（セクション6で毎回同期）。セクション4 ゲート（1 回目・2 回目）と振る舞いチェックも省略不可。
- 「最近触っていない」「別 PR で扱う」「別ブランチに任せる」を根拠に backlog から項目を外さない（[`no-convenience-tech-debt.mdc`](../../../rules/no-convenience-tech-debt.mdc)）。
- 最終洗い出しから時間が経過した、別ブランチで運用していた等で鮮度に疑いが出たら、空到達を待たずに洗い出しを当て直してから運用を継続する。
- backlog が空に見えるだけで裏取りを省略して外側ループ終了を宣言しない。

## GCP（本番を断定するとき）

- 本番の現在状態（デプロイ済みリビジョン、リソース設定、該当ログの有無など）を断定するときは、gcloud / Cloud Logging 等で確認するか、未確認と明示する。
- 一般的な GCP の仕様、設計パターン、ローカル再現手順は観測不要。
- 「たぶん本番も同じバグ」と言いたいときは、断定せず未確認か観測のどちらかに寄せる。

権限の有無（gcloud が使えること）と上記の主張タイプを混同しない。

## シェル完了まで結果を断定しない

[`dont-finish-task-while-process-is-running.mdc`](../../../rules/dont-finish-task-while-process-is-running.mdc) に従う（手順は [`process-monitor`](../../process-monitor/SKILL.md)）。

## チャットと手順書の役割分担

- ユーザー向けチャット: ワークフローの番号を主語にしない（意味語でよい）。
- リポジトリ内の手順書（`SKILL.md` 等）: セクション番号アンカーを維持する。
