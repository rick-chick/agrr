# 要件仕様書テンプレート

## 概要

このテンプレートは、要件分析フェーズの成果物をまとめた仕様書を作成するための標準フォーマットです。プロジェクト調査、要求理解、要求ブラッシュアップの結果を構造化して文書化します。

## テンプレート構造

### 1. 仕様書ヘッダー

```markdown
# 要件仕様書: [機能名/システム名]

**作成日**: [YYYY-MM-DD]
**作成者**: [作成者名]
**バージョン**: [1.0]
**ステータス**: [draft | review | approved | implemented]

## 概要

[機能/システムの目的とスコープの簡潔な説明]

## 背景・目的

- **ビジネス目的**: [なぜこの機能が必要なのか]
- **ユーザー価値**: [ユーザーにどのような価値を提供するか]
- **期待効果**: [導入による効果・改善点]

## 対象ユーザー

- **主要ユーザー**: [この機能を使う主なユーザー層]
- **関係者**: [影響を受ける他のユーザーやシステム]
```

### 10. レビューポイント

```markdown
## レビューポイント

### 要件の完全性
- [ ] すべてのステークホルダーの要求を反映しているか
- [ ] 受け入れ条件が明確に定義されているか
- [ ] 優先度付けが適切か

### 要件の妥当性
- [ ] 技術的に実現可能か
- [ ] ビジネス価値が明確か
- [ ] 既存システムとの整合性が取れているか

### 要件のテスト容易性
- [ ] 受け入れ条件がテスト可能か
- [ ] 曖昧な表現がないか
- [ ] 測定可能な基準が設定されているか
```

## 使用方法

1. このテンプレートをコピーして新しい仕様書を作成
2. プロジェクト調査の結果を基に各セクションを埋める
3. 要求理解の結果を機能要件・非機能要件に反映
4. 要求ブラッシュアップの結果を制約・リスクに反映
5. レビューポイントを確認しながら仕様を洗練

## 注意事項

- 要件は必ず測定可能・テスト可能な形で記述する
- 曖昧な表現（例: 「高速」「使いやすい」）は避ける
- すべての要件に優先度と見積もりを付与する
- ステークホルダー全員の合意を得てから承認する
