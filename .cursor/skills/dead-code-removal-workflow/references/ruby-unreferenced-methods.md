# Ruby / Rails — 単一静的探索で落ちる未到達の補完

Phase A の **`rg` 横断**や **`debride app lib config` のみ**では、次の形のコードが **未到達・テスト専用表面**として検出落ちする。本稿はその **補完手順**を固定する（断定は常に意味読みと動的経路確認のあと）。

## 検出落ちの典型（なぜ足りないか）

| パターン | 起きること |
|----------|------------|
| **ポートと実装の同名** | `lib/domain/**/gateways/*.rb` の宣言行とアダプタの `def` 行にだけシンボルが出て、「呼び出しがある」ように見える。インタラクタ等に **実呼び出しが無い**と気づきにくい。 |
| **`module_function` の複数エントリ** | モジュール内の **メソッド A は本番**、**メソッド B は同一ファイル＋テストのみ**など、**メソッド単位**で切らないと見落とす。 |
| **インスタンス名の `rg`** | `\.foo\b` は **別クラスの `foo`** と衝突する。逆に **短い名前**はノイズが多い。 |
| **`debride`（テストディレクトリを渡さない場合）** | テストからしか呼ばれない実装は **候補に上がりうる**が、Rails の **ルーティング・コールバック・`send`** は未解析のため **偽陽性**も多い。**機械出力を根拠に単独で削除しない。** |
| **未定義定数への参照** | メソッドが一度も実行されていないと、`SomeEntity` が **未定義のまま**残る。テキスト検索だけでは「デッドか未配線か」判別しづらい。 |

## 補完手順（Phase A 内で実施）

1. **シンボルの全文検索**  
   `test/` を含む。識別子は **`'ExactMethodName'`** と、必要なら **`\.method_name\b`** など複数パターンを試す。定義行と **宣言行（NotImplementedError）** を呼び出しと区別する。

2. **呼び出し側の有無（ファイルを開く）**  
   インタラクタ、ゲートウェイ利用箇所、`CompositionRoot`、`config/` のジョブ・initializer、ルート境界を確認する。**`@gateway.method` のような実呼び出し**が無ければ未到達候補。

3. **ポートと実装の同名**  
   ドメインの抽象メソッド名とアダプタの `def` が一致する場合、**宣言＋実装の 2 ファイル以外にその識別子が現れない**かを確認する。現れなければ **ポート表面の冗長**または **実装デッド**の候補。

4. **`module_function` / クラスメソッド**  
   **メソッド名ごと**に参照ファイルを列挙する。本番パス（`app/` `lib/` のうち `test/` 以外）に **呼び出しファイルが無い**が `test/` にだけある → **テスト専用表面**候補。

5. **同一クラス内の private**  
   シンボルが **定義ファイルの `def` のみ**なら、同一ファイル内の **`self.` 呼び出し**、`method(:sym)`、**`send` / `public_send`** を読んで回す。それでも無ければ **未到達 private** 候補。

6. **定数の実在**  
   メソッド本体が `Domain::...::SomeEntity.new` 等を参照する場合、`lib/domain/**/entities/` 等に **定義ファイルがあるか**、または Rails 起動後に **`const_defined?` が真か** を確認する。**参照先が存在しない**のにメソッドが未到達なら、削除優先度が高い（ただし Zeitwerk の自動読み込みパスも確認する）。

## 区分（削除は Phase C の確定後）

| 区分 | 扱い |
|------|------|
| **リポジトリ全体未到達** | 動的経路・ルート・ジョブを再確認後、デッドとして削除候補。 |
| **テストのみから参照** | 本番契約に不要なら実装とテストをまとめて削除。必要なら本番配線または契約更新。 |
| **ポートのみに残る表面** | 実装・呼び出しとも無ければポートから削除し、実装側のデッド `def` も削除。 |

## 禁止（本スキル全体の再掲）

- **`rg` のヒット数だけ**でデッドと断定しない。  
- **`debride` の行だけ**でデッドと断定しない（Rails 未解析の偽陽性・テスト呼び出しの取りこぼしの両方ありうる）。

詳細な `rg` 例は [CHECKLIST.md](CHECKLIST.md) を用いる。
