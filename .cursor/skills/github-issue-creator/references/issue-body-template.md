# Issue 本文テンプレート

UX/UI 監査由来は **`ux-issue-creator`** のテンプレートを使う。

## バグ

```markdown
## 再現手順

<起票前に実施済み。手順・入力・環境>

## 期待する動作

<…>

## 実際の動作

<…>

## 完了条件

- [ ] <観測可能>
- [ ] test-common GREEN

## 依存

- なし

## 参照

- コード: `path/to/file`
- 関連 issue: #N
```

## 調査（未再現バグ）

```markdown
## 観測事実

<ログ・画面・環境。再現に足りない情報も明記>

## 試した再現手順

<実施済み手順と結果。未再現の理由が分かれば記載>

## 完了条件

- [ ] 再現手順が確定し、バグ issue へ昇格できる
- [ ] または再現不能の根拠が観測可能

## 依存

- なし

## 参照

- コード: `path/to/file`
```

## 機能・改善

```markdown
## 目的

<現状と望ましい振る舞い。起票前に確認済み。観測可能に>

## 完了条件

- [ ] <観測可能>
- [ ] test-common GREEN

## 依存

- なし

## 参照

- コード: `path/to/file`
- 関連 issue: #N
```

## タイトル

```
[P0|P1|P2] <要約>
[P0|P1|P2][区分] <要約>   # 区分任意: bug, feat, i18n, UX, refactor, infra
```

| 優先度 | 目安 |
|--------|------|
| P0 | 主要フロー破綻・データ不整合・セキュリティ |
| P1 | ユーザー影響あり |
| P2 | 改善・負債 |

## ラベル

| ラベル | 条件 |
|--------|------|
| `bug` | 再現済みの不具合 |
| `enhancement` | 機能改善 |
| `documentation` | ドキュメントのみ |
| `agent-ready` | Worker 即着手（起票後付与） |

## 起票前チェック

- [ ] バグは再現済み
- [ ] 完了条件が観測可能
- [ ] 修正方法を書いていない
- [ ] 1 issue = 1 スコープ
- [ ] 重複確認済み
